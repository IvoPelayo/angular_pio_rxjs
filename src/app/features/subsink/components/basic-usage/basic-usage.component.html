Problema

Ahora bien, esto suena bastante engorroso... Si bien nuestros componentes deberían ser lo más atomizados y simples, es posible que en un componente tengamos múltiples suscripciones, lo cual me llevaría a declarar una variable por cada suscripción, para luego dentro del ciclo de ngOnDestroy *verificar* que esta variable contiene una suscripción, y finalizarla.

Solución: Subsink

Con esta pequeña librería obtenemos una clase que maneja suscripciones de manera sencilla, permitiendo finalizarlas en solo una linea de código.
